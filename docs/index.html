How it works

1

Schedule fires overnight

A GitHub Actions workflow runs on cron. Curator reads your config.yml, checks for a kill switch, then scans the configured paths — lib/, components/, app/, content/ by default.

2

Each finding is LLM-verified

Automated tools flag candidates across all enabled scopes. An LLM then filters false positives and assigns a confidence rating — high, med, or low. Findings below your threshold are silently skipped.

3

Issues land in your repo

Up to 5 labelled issues per run, each with a finding, evidence, suggested action, and confidence rating. Deduplication prevents the same issue being filed twice. You decide what to act on.

Detection scopes

dead-exports

Exported symbols with no references elsewhere in the codebase — safe to remove or unexport.

oversized-files

Files that have grown past the line or symbol threshold and likely need splitting into smaller modules.

stale-placeholders

TBD, Coming soon, lorem ipsum, and year markers old enough to be embarrassing. Flags content rot before it ships.

Install

bash 
# 1. Add Curator as a submodule
git submodule add https://github.com/derrybirkett/curator .curator

# 2. Copy the config to your repo root
cp .curator/config.yml .curator-config.yml

# 3. Add your Anthropic API key to GitHub repo secrets
#    Settings → Secrets → ANTHROPIC_API_KEY

# 4. Commit and push — Curator runs at 3am UTC
git add .curator .github/ .curator-config.yml .gitmodules
git commit -m "chore: install curator"
git push

Configure

Everything lives in .curator-config.yml in your repo root. Flip dry_run off once you're happy with the findings.

dry_run: true # writes to workflow summary only — no issues filed max_issues_per_run: 5 min_confidence: med # high | med | low dedup_window_days: 30 scopes: dead_exports: enabled: true threshold: 3 oversized_files: enabled: true line_threshold: 400 stale_placeholders: enabled: true year_marker_cutoff: 2

Kill switch: create .curator-pause at your repo root to halt all runs immediately.
